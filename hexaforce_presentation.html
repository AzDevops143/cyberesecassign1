Hexa Force Architecture

Anatomy of a Container Escape vs. Surgical Containment Defenses

1. The Kernel Exploit

The attacker triggers the Dirty COW race condition (CVE-2016-5195) to overwrite a read-only root file directly from the unprivileged container.

sequenceDiagram
    participant C as Container
    participant Mem as /proc/self/mem
    participant K as Host Kernel
    C->>K: mmap() read-only file
    par madvise thread
        loop 
            C->>Mem: madvise(MADV_DONTNEED)
        end
    and write thread
        loop
            C->>Mem: write(payload)
        end
    end
    Note over Mem,K: Kernel race condition occurs
    K-->>K: Writes to underlying physical file!
victim@container:~$ ./hexaforce_cow --exploit
[!] Overwriting /usr/bin/passwd...
[+] Container root achieved.

Defense: Seccomp Filtering

We deploy a surgical eBPF Seccomp profile to block the specific madvise syscall required for the race condition, stopping it at the kernel boundary.

"names": ["madvise"],
"action": "SCMP_ACT_ERRNO"

2. Process Injection

Using shared PID namespaces, the attacker abuses CAP_SYS_PTRACE to inject malicious shellcode directly into host daemons.

[!] Scanning host PID namespace...
[+] Injecting payload into PID 1204 (Host)

Defense: Capability Dropping

By enforcing the Principle of Least Privilege, we mathematically eliminate process injection capabilities from the container environment.

docker run --cap-drop=ALL \
           --cap-drop=SYS_PTRACE

3. Daemon API Hijacking

The attacker accesses a mounted /var/run/docker.sock, gaining raw HTTP access to the Docker engine to manipulate infrastructure.

curl -X POST --unix-socket /var/run/docker.sock
[+] Spawning unconstrained sibling container

Defense: Socket Isolation

We strictly prohibit exposing the Docker daemon socket to unprivileged containers, completely neutralizing API hijacks.

Rule: Avoid mounting -v /var/run/docker.sock

4. Persistent Takeover

The attacker mounts the physical host filesystem (/) and writes a reverse shell to the host's cron daemon.

echo "* * * * * root reverse_shell" > /mnt/host/etc/cron.d/pwn
[+] Physical host compromised persistently.

Defense: Immutable Mounts

We enforce read-only filesystems and strict volume controls, making it physically impossible for attackers to drop persistent backdoors.

docker run --read-only \
           --security-opt no-new-privileges
