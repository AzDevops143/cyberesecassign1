How The Attack Happens

An attacker breaks out of a container and takes over the physical host in four cascading stages.

1. The Kernel Exploit

The attacker runs
madvise() and writes to /proc/self/mem simultaneously, triggering the Dirty COW race condition to overwrite read-only host memory.

2. Process Injection

Because the container shares the host's PID namespace, the attacker uses CAP_SYS_PTRACE to inject malicious shellcode directly into a running host process.

3. Daemon Hijacking

The attacker accesses a poorly configured /var/run/docker.sock mount, giving them raw HTTP API access to the host's Docker engine.

4. Host Persistence

Using the hijacked API, they spawn a new container that mounts the host's root filesystem (/), writing a backdoor directly into /etc/cron.d.

How We Stop The Attack

Hexa Force implements an 8-layer defense-in-depth architecture that surgically severs the exploit chain without heavy performance penalties.

1. Seccomp Filtering

We deploy a custom eBPF Seccomp profile that strictly blocks the madvise system call. This instantly neutralizes the Dirty COW memory race condition at the kernel boundary.

2. Capability Dropping

We explicitly strip CAP_SYS_PTRACE and CAP_SYS_ADMIN from the container. Without these Linux capabilities, process injection becomes mathematically impossible.

3. Namespace Isolation

We enforce strict PID namespace isolation. The container can no longer "see" host-level processes, preventing targeting and lateral movement.

4. Immutable Mounts

We restrict volume mounts and enforce read-only root filesystems (--read-only), ensuring the host filesystem cannot be tampered with even if API access is compromised.
